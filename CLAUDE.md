# Amazon SES API経由メール送信機能のサンプルプロジェクト

- このプロジェクトはAmazon SESを使って、外部の業務システムからAPI経由で通知メールを送信する仕組みを試すためのサンプルプロジェクトです。
- API Gateway + Lambda + SES の構成で、自由形式メールとSESテンプレートメールの両方をサポートします。
- DNS設定不要・SES Sandbox（メールアドレス検証のみ）で動作するため、手軽に試せます。
- 対象ユーザーはAWSクラウドプラクティショナーを取得しているレベルを想定します。
- リソースの作成はなるべくAWS CDKでのIaCでの構築を基本とします。どうしてもCDKで構築できない場合はAWS CLIコマンドで操作を行います。なるべくマネジメントコンソールでの操作は行いません。
- AWSの認証情報は環境変数に埋め込まれたデフォルトの認証情報を使う場合とAWS SSOの一時的なセッショントークンによる認証の2つを想定してドキュメントを構築します。実際のコードでもこの2つを網羅するようにする必要があります。
- なるべく cdk.json に設定を入れて、コマンドはなるべくシンプルにします。そのため、デフォルトリージョンもcdk.jsonに書いている、そういう背景があります。
